1. Problem Statement

Many people earn money every month but do not know where it is being spent. They often forget small daily expenses, which makes it difficult to understand their overall spending habits and savings.

Although tools like Excel or Notes can be used to record expenses, they require manual calculations and organization. This application will provide an easier way to record income and expenses, organize transactions, and view monthly, quarterly, and yearly spending summaries.

2. Target Users

This application is designed for people who want to manage their personal finances.

The main target users are:

Salaried employees
Students
Freelancers
Anyone who wants to track their income and expenses

The goal is to help users understand how they spend their money and improve their financial habits.

3. Goals

The main goals of this application are:

Help users record their income and expenses.
Show users where their money is being spent.
Allow users to view their financial history.
Provide monthly, quarterly, and yearly summaries.
Help users keep track of their savings and spending.
4. Functional Requirements (MVP)

For Version 1, the application should allow users to:

Register a new account.
Log in securely.
Add one or more income records.
Add expense records.
Create and manage expense categories.
View all transactions in chronological order, similar to a bank statement.
Edit and delete income and expense records.
View a dashboard with spending summaries.
Log out securely.
5. Non-Functional Requirements

The application should:

Respond quickly to user requests.
Store passwords securely using password hashing.
Allow users to access only their own data.
Protect user information from unauthorized access.
Be easy to maintain and extend with new features in the future.
6. Out of Scope (Version 1)

The following features will not be included in the first version:

AI spending insights
Budget planning
Recurring transactions
Email notifications
Multi-currency support
Export to PDF or Excel
Family or shared accounts

These features can be added in future versions.

7. Success Criteria

The first version of the application will be considered successful if:

Users can register and log in successfully.
Users can add, edit, and delete income and expense records.
Users can organize expenses using categories.
Users can view transaction history.
Users can see monthly spending summaries.
Each user can access only their own financial data.
